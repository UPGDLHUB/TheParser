# Parser

A draft for a **Java-based predictive recursive-descent parser** that processes a  sequence of tokens generated by a lexer and validates the syntactic structure of a simple programming language. It reads the **token vector produced by the lexer** and verifies that the input program follows the defined grammar.

# Grammar Overview

```
CLASE                -> "class" identifier "{" DECLARACION_VARIABLE* "}"

DECLARACION_VARIABLE -> TIPO identifier ("=" EXPRESION)? ";"

TIPO                 -> entero | int | char | cadena | string | booleano | boolean

EXPRESION            -> LITERAL

LITERAL              -> integer | character | string
```

# Running the Parser

1.  The lexer scans the input file and produces tokens.
2.  The parser reads the token vector.
3.  The parser validates the syntactic structure.

Example output:

    rule_clase (root)
    -rule_declaration_variable
    -rule_expression
    -rule_literal
    Parsing completed successfully.


# Maven Dependencies

⚠️ **Note:** The lexer used by this project is provided by the **TULIP library**, which is distributed through **JitPack**. Therefore, the project must include the JitPack repository and the TULIP dependency in the `pom.xml`. You need the following configuration to your `pom.xml`:

```xml
<repositories>
    <repository>
        <id>jitpack.io</id>
        <url>https://jitpack.io</url>
    </repository>
</repositories>

<dependencies>
    <dependency>
        <groupId>com.github.javiergs</groupId>
        <artifactId>TULIP</artifactId>
        <version>v7.0</version>
    </dependency>
    <!-- other dependencies here -->

</dependencies>
